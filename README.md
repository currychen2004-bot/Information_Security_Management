# Information Security Management 课程项目说明

## 1. 项目概述

本仓库用于汇总《Information Security Management》课程中的成员个人项目与配套实验环境。整体内容分为两部分：

- `成员代码/`：每位成员独立完成的课程小项目，方向覆盖图像分类、数据投毒、SQL 注入检测、DVWA 暴力破解、图像隐写分析以及一个基于 Flask 的桌面宠物待办应用。
- `漏洞靶场/`：用于安全实验的 DVWA 靶场环境，供 SQL 注入、暴力破解等脚本在受控条件下测试。

仓库的组织方式更偏向课程成果归档，而不是单一应用工程，因此阅读时建议先从目录入手，再分别进入各成员项目查看 README 和主程序。

## 3. 各部分功能说明

### 3.1 成员代码

#### 1. `CNN 图像分类-孙昱`

项目目标是使用一个基础卷积神经网络完成 MNIST 手写数字分类。

主要文件：

- `main.py`：整个分类流程的入口文件，负责设置随机种子、下载数据集、构建 CNN、训练模型、测试模型并输出样例预测结果。
- `README.md`：说明环境依赖、运行方式和实验流程。

代码功能拆分：

- `SimpleCNN`：定义两层卷积和全连接分类器。
- `set_seed()`：固定随机种子，减少实验结果波动。
- `train_one_epoch()`：执行单轮训练并返回训练损失与准确率。
- `evaluate()`：在测试集上计算损失和准确率。
- `show_predictions()`：打印若干测试样本的预测结果。
- `main()`：组织数据加载、模型训练和结果输出。

#### 2. `CNN数据投毒攻击-肖熠`

项目目标是针对 MNIST 训练过程模拟数据投毒攻击，观察模型在被污染标签训练后的性能变化。

主要文件：

- `main.py`：定义 AlexNet 变体、投毒数据构造、模型训练、准确率与损失曲线绘制、错误分类样本展示。
- `README.md`：介绍实验目的、投毒机制和效果展示。

代码功能拆分：

- `AlexNet`：为 28x28 的 MNIST 图像改写后的卷积网络。
- `select_subset()`：从完整数据集中随机抽取子集，加快实验速度。
- `fetch_datasets()`：按设定比例向训练集注入错误标签，构造投毒样本和干净样本。
- `plot_classified_images()`：展示模型在测试集上的正确或错误分类样本。
- 主程序部分：完成数据准备、投毒训练、精度统计和 Matplotlib 可视化。

#### 3. `DVWA SQL 注入简易扫描工具-刘佳仑`

项目目标是对 DVWA 的 SQL 注入页面做一个基于规则的快速探测脚本。

主要文件：

- `dvwa_sqli_scanner.py`：扫描主程序。
- `README.md`：介绍使用前准备、参数说明、输出结果和局限性。

代码功能拆分：

- `parse_cookie()`：将浏览器复制出的 Cookie 字符串转成字典。
- `build_cookie_header()`：把 Cookie 字典拼成请求头。
- `split_url_and_params()`：解析目标 URL 与查询参数。
- `send_get()`：发送 GET 请求并返回状态码、正文和耗时。
- `has_sql_error()`：匹配常见 SQL 报错特征。
- `scan_sqli()`：对每个参数逐个拼接 payload，依据报错、响应长度变化和延时特征输出可疑点。
- `format_findings()`：将扫描结果格式化输出。
- `main()`：接收命令行参数并驱动扫描流程。

#### 4. `DVWA 暴力破解工具-何其杰`

项目目标是针对 DVWA Brute Force 模块做一个多线程字典爆破脚本。

主要文件：

- `dvwa_brute.py`：暴力破解主程序。
- `username.txt`：用户名候选字典。
- `password.txt`：密码字典。
- `README.md`：说明使用方式和实验要求。

代码功能拆分：

- `BruteForcer.__init__()`：初始化目标地址、Cookie、用户名、密码字典和线程数。
- `load_tasks()`：加载用户名与密码组合并放入任务队列。
- `worker()`：线程执行函数，循环发起请求并判断是否破解成功。
- `run()`：创建线程并等待所有任务结束。
- `main()`：解析命令行参数，支持单用户名和用户名文件两种模式。

#### 5. `数字图像隐写分析-董璟阳`

项目目标是演示图像信息隐藏与分析，包括 LSB 信息嵌入、提取、PSNR 评估、位平面分析和 DCT 频域失真模拟。

主要文件：

- `main.py`：图像预处理、隐写、提取、分析与可视化主程序。
- `README.md`：介绍实验原理、输入文件要求和运行步骤。

代码功能拆分：

- `get_binary_msg()`：把字符串转换成二进制序列。
- `psnr()`：计算嵌入前后图像的峰值信噪比。
- `preprocess_image()`：将输入图像灰度化并缩放到固定尺寸。
- `lsb_encode()`：把秘密信息逐位写入图像像素最低位。
- `lsb_decode()`：从隐写图像中提取秘密信息。
- `bit_plane_analysis()`：展示不同位平面对图像内容的影响。
- `dct_distortion_demo()`：进行 DCT 变换、频域截断和 IDCT 重建演示。

#### 6. `桌面宠物代办清单-陈柯睿`

项目目标是实现一个轻量的桌面宠物待办应用，用待办完成情况驱动宠物状态变化。

主要文件：

- `app.py`：Flask 后端，负责页面渲染、任务数据读写与接口提供。
- `templates/index.html`：页面结构模板。
- `static/styles.css`：界面样式与宠物动画。
- `static/script.js`：前端请求和页面重渲染逻辑。
- `data/state.json`：本地任务状态数据文件。
- `README.md`：项目单独说明文档。

代码功能拆分：

- `load_state()` / `save_state()`：读取和保存 JSON 状态。
- `level_index()`：根据成长值计算宠物等级。
- `build_payload()`：统一组装前端需要的待办数据和宠物状态数据。
- `/api/state`：返回当前应用状态。
- `/api/todos`：新增待办。
- `/api/todos/<todo_id>`：修改或删除指定待办。
- `/api/todos/clear-done`：清空已完成任务。
- `/api/pet/pat`：返回“摸摸它”后的宠物反馈。

### 3.2 漏洞靶场

#### `漏洞靶场/DVWA`

该目录为 DVWA 靶场环境，主要用于安全课程中的漏洞复现与工具测试。该部分并非本仓库成员自行从零编写，而是引入的现成靶场项目，仓库中保留它的作用主要是为了：

- 为 SQL 注入扫描与暴力破解脚本提供受控测试对象。
- 支持课程演示和实验复现。
- 让成员项目在同一仓库内完成环境配套。

常见入口文件和配置文件包括：

- `compose.yml`：容器编排文件。
- `Dockerfile`：镜像构建说明。
- `index.php`、`login.php`、`setup.php`：DVWA 页面入口和初始化页面。
- `README.md`、`README.zh.md`：DVWA 官方项目说明。

## 4. 运行建议

由于仓库内包含多个独立项目，建议按子目录分别运行，不建议把整个仓库视为单一 Python 工程。

### Python 类项目

进入对应成员目录后，按各自 README 安装依赖并执行主程序，例如：

```bash
cd "成员代码/CNN 图像分类-孙昱"
python main.py
```

或：

```bash
cd "成员代码/桌面宠物代办清单-陈柯睿"
python app.py
```

### DVWA 相关项目

SQL 注入扫描和暴力破解脚本依赖本地 DVWA 靶场，使用前应先完成：

1. 启动 DVWA。
2. 登录并把安全级别设置为 `Low`。
3. 从浏览器复制当前 Cookie。
4. 再运行对应脚本。
